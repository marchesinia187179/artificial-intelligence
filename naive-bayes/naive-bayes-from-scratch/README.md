# Gaussian Naive Bayes Classifier (Log-Space Implementation)

This repository provides a numerically stable, robust implementation of the **Gaussian Naive Bayes** algorithm built from scratch using `NumPy`. Unlike basic implementations, this version utilizes **log-space arithmetic** to prevent numerical underflow, making it suitable for high-dimensional datasets.

## Key Features
* **Numerical Stability**: Uses `np.log` to transform probability multiplications into additions.
* **Gaussian PDF**: Implements the standard normal distribution formula for continuous features.
* **Epsilon Smoothing**: Includes a small variance offset (`1e-9`) to prevent division by zero errors.

## Mathematical Core

### 1. Training (`fit` method)
The model computes statistical parameters for each class:
* **Priors**: The probability of each class occurring.
* **Means**: The average of each feature.
* **Variance**: The spread of each feature, with added epsilon for stability.

### 2. Likelihood and Prediction
Instead of multiplying probabilities (which leads to vanishingly small numbers), the model sums the logs of probabilities:

$$\text{Posterior} \propto \log(P(y)) + \sum \log(P(x_i | y))$$

The `_pdf` method implements the Gaussian density function:

$$f(x) = \frac{1}{\sqrt{2\pi\sigma^2}} \exp\left(-\frac{(x-\mu)^2}{2\sigma^2}\right)$$

## Code Structure

### The `NaiveBayes` Class
```python
def _pdf(self, class_idx, x):
    mean = self._mean[class_idx]
    var = self._var[class_idx]
    numerator = np.exp(-((x - mean) ** 2) / (2 * var))
    denominator = np.sqrt(2 * np.pi * var)
    return numerator / denominator
```

## Why use Log-Space?
In classification tasks with many features, the probability of a data point belonging to a class can become extremely small (e.g., $10^{-50}$). Computers struggle to represent these numbers accurately, often leading to numerical underflow. 

By using the log, we transform the product of probabilities into a sum:
$$\log(P(x_1) \cdot P(x_2)) = \log(P(x_1)) + \log(P(x_2))$$

This expands the range of representable values, ensuring the model remains accurate and reliable regardless of feature count.
